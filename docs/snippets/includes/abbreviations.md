*[HPC]: High-Performance Computing — using clusters of many computers for large workloads
*[CPU]: Central Processing Unit
*[GPU]: Graphics Processing Unit — accelerates AI/ML and some simulations
*[RAM]: Random Access Memory — a job's fast working memory
*[I/O]: Input/Output — reading and writing data
*[SU]: Service Unit — the accounting unit for cluster usage
*[SUs]: Service Units — the accounting unit for cluster usage
*[SMP]: Symmetric Multi-Processing — single-node, shared-memory cluster
*[MPI]: Message Passing Interface — for multi-node parallel jobs
*[HTC]: High-Throughput Computing — many independent single-node jobs
*[OOD]: Open OnDemand — the browser-based access portal
*[SSH]: Secure Shell — protocol for connecting to a login node
*[VPN]: Virtual Private Network — required to reach the clusters
*[CLI]: Command-Line Interface
*[GUI]: Graphical User Interface
*[Lmod]: The module system used to load software
*[Slurm]: The workload manager (scheduler) used by CRCD
*[CUDA]: NVIDIA's platform for programming GPUs
*[X11]: The system that forwards graphical windows over SSH
*[VIZ]: The graphical Linux desktop environment
*[TEACH]: The cluster reserved for courses and teaching
*[Resource Allocation]: An allotment of computing time and/or data storage quota
*[PittNet]: The internal University network
*[CRCD Ecosystem]: The complement of CRCD infrastructure
